# ADR 003: Address Normalization Strategy
## Status

Accepted

## Decision

Normalize delivery addresses before performing comparisons or fraud analysis.

## Context

Address data frequently contains:

- Formatting differences
- Capitalization inconsistencies
- Accents and special characters
- User-entered variations

These inconsistencies make exact matching unreliable.

## Decision Rationale

Address normalization includes:

- Upper-case conversion
- Character translation
- Punctuation cleanup
- Whitespace normalization

The goal is not perfect address matching but consistent comparison.

## Consequences

### Positive

- Improved address correlation
- Better pickup-location matching
- More reliable historical analysis

### Negative

- Requires maintenance of normalization rules
- Some edge cases may still require manual review

## Alternatives Considered

### Exact Matching

Rejected because it misses many equivalent addresses.

### Fuzzy Matching Algorithms

Rejected due to additional computational cost and operational complexity.

## Outcome

Normalized addresses became the standard representation used throughout the fraud-detection process.
